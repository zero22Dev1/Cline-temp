# PDF変換方針

## 形式の選択

| 条件 | 推奨形式 |
|---|---|
| 通常の文章、検索、要約、仕様抽出 | Markdown |
| 表、段組み、視覚的な階層をブラウザーで確認 | HTML |
| 検索と視覚確認の両方が必要 | both |
| 画像のみのスキャンPDF | OCR後にMarkdown。OCR未実施なら`OCR_REQUIRED` |

HTMLもテキスト抽出を元にするため、原本レイアウトの完全再現ではない。表、図、数式、脚注が要件判断に影響する場合は、対象ページを画像化して原本と照合する。

## コンテキスト効率

- 最初に`index.md`だけを読む。
- 検索結果からページ範囲を特定する。
- 1回に読むチャンク数を現在の判断に必要な範囲へ限定する。
- 読み終えた詳細は、ページ参照付きの短いCheckpointへ圧縮する。
- 全文要約を正本にせず、原本ページへ戻れる状態を保つ。

## 抽出状態

- `OK`: 1ページあたり100文字以上を抽出
- `LOW_TEXT`: 1から99文字。図表中心または抽出不足の可能性
- `OCR_REQUIRED`: 文字抽出が0。空白ページか画像ページかを原本で確認
- `EXTRACTION_ERROR`: ライブラリがページを処理できなかった
